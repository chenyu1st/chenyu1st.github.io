Ubuntu16.04安装opencv
发表于 2020-12-28| 更新于 2020-01-07|OpenCV
字数总计:1.9k|阅读时长: 9 分钟
| 阅读量:|评论数:

声明:这是我在大学毕业后进入第二家互联网公司学习的内容

背景

公司有图像处理的项目需要安装opencv，基本都是C相关的组件（我的盲区），开始学习下opencv相关知识

简单了解

Conda

Conda是一个开源跨平台语言无关的包管理与环境管理系统。由“连续统分析”（Continuum Analytics）基于BSD许可证发布。

Conda允许用户方便地安装不同版本的二进制软件包与该计算平台需要的所有库。还允许用户在不同版本的包之间切换、从一个软件仓库下载包并安装。

Conda是用Python语言开发，但能管理其他编程语言的项目（如R语言），包括多语言项目。 Conda可安装Python语言的包，类似于其他基于Python的跨平台包管理器（如wheel 页面存档备份，存于互联网档案馆或pip）。

opencv

opencv的全称是：Open Source Computer Vision Library

opencv是Intel开源计算机视觉库。它由一系列 C 函数和少量 C++ 类构成，实现了图像处理和计算机视觉方面的很多通用算法。

准备

  • 操作系统：ubuntu16.04
  • opencv版本：3.3.1
  • conda版本：4.8.2
  • python版本：3.7.6

下载与安装

安装Conda及Python

wget https://repo.anaconda.com/archive/Anaconda3-2020.02-Linux-x86_64.sh
chmod +x Anaconda3-2020.02-Linux-x86_64.sh
#安装时已经自动添加到环境变量中
#export PATH=$HOME/anaconda3/bin/:$PATH
# 安装 python 3.7.6
conda install python=3.7.6

编译

注意：不要随便执行make -j，可能会把服务器跑挂

参考你自己机器点CPU和内存大小去选择-j的数量

编译失败需要build目录执行make clean 再cmake和make

或者直接删除build目录，再重头来遍

cd opencv-3.3.1/build/
cmake -D CMAKE_BUILD_TYPE=RELEASE -D  CMAKE_INSTALL_PREFIX=/home/ubuntu/anaconda3 -D CMAKE_CXX_FLAGS:STRING=-fpermissive  INSTALL_PYTHON_EXAMPLES=ON  -D INSTALL_C_EXAMPLES=OFF  -D  PYTHON_EXCUTABLE=/home/ubuntu/anaconda3/bin/python -D WITH_TBB=ON -D WITH_V4L=ON   -D WITH_GTK=ON  -D WITH_OPENGL=ON -D BUILD_EXAMPLES=ON ..

make -j2

安装及动态库链接

sudo make install
sudo ldconfig

安装eigen

sudo apt-get install libeigen3-dev

报错总结

pip报错

不管是执行pip还是pip3 install 或者-v 都返回下面的错误

Traceback (most recent call last):
File "/home/myuser/.local/bin/pip", line 7, in <module>
from pip._internal import main
ImportError: No module named 'pip._internal'

解决方法

python2 -m pip install --user --upgrade pip

python3 -m pip install --user --upgrade pip

cmake缺包

-- checking for module 'gstreamer-base-1.0'
--   package 'gstreamer-base-1.0' not found
-- checking for module 'gstreamer-video-1.0'
--   package 'gstreamer-video-1.0' not found

解决方法

apt-get install 包名

注意有些包的包名不一定是报错的包名

例如：

“libavresample libgphoto2”
的实际安装包名为
“libavresample-dev libgphoto2-dev”

执行

locate libopencv_core.so.3.3

看样子matlab找不到lib包的位置，而不是这些包不存在

解决方案

软链接报错的库到系统库里

sudo ln -s /home/ubuntu/opencv-3.3.1/build/lib/libopencv_core.so.3.3 /usr/local/lib/libopencv_core.so.3.3

/lib/x86_64-linux-gnu/libz.so.1: version `ZLIB_1.2.9’ not found

缺少ZLIB_1.2.9的包

解决方案

下载ZLIB_1.2.9的包

Download

tar -xvf ~/Downloads/zlib-1.2.9.tar.gz
cd zlib-1.2.9
sudo -s
./configure; make; make install
cd /lib/x86_64-linux-gnu
ln -s -f /usr/local/lib/libz.so.1.2.9/lib libz.so.1
cd ~
rm -rf zlib-1.2.9

总结

兜兜转转，花了2天才安装完opencv，主要问题在于编译的报错，出现各种莫名其妙的错误

而且相同机器（配置、版本、文件路径全部一样）报错能不一样，我人傻了

一共安装了4台，感觉就是折磨

由于这台机器上不止需要部署opencv，还有很多其他的C组件，而组件之间的版本依赖及服务依赖很强

后期我需要好好理一下之间的关系，将这台机器的服务docker化
